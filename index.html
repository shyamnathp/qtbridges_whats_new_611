What is QtBridge for Python?

Problem: Exposing Python objects to QML today requires significant PySide6 boilerplate — @Property, @Signal, @Slot, QAbstractItemModel subclasses…

Solution: QtBridge wraps any plain Python class and exposes it to QML automatically. No Qt knowledge required.

You write QtBridge does
self.count = 0 in __init__ Full Qt property + change signal
def data(self) -> list[dict] QML-ready table model
Any public method Callable slot from QML
QtBridge is a framework that takes any plain Python class and connects it to QML without boilerplate. Auto-properties: self.count = 0 becomes a full Qt property with a change signal. Table data: return a list of dicts, dataclasses, or a Polars DataFrame from data() — QtBridge handles the rest. Public methods become QML-callable slots automatically.

Install & Setup

Install from PyPI (once published)

pip install qtbridge

Build from source (clone + limited-API wheel)

git clone git://code.qt.io/qt/qtbridge-python.git
cd qtbridge-python
pip wheel . -Ccmake.define.QT_PATHS=/path/to/qtpaths

Build the documentation (API + examples)

cd docs && pip install -r requirements.txt
make html
1. pip install qtbridge — once we're on PyPI, this is all anyone needs. 2. Building from source: git clone then pip wheel with QT_PATHS pointing to your Qt installation. Limited-API build, non-debug. 3. Docs: cd docs, pip install requirements, make html. Opens in docs/_build/html/index.html.

user_dataset shows the same QML table backed by three different Python data sources. Dict variant: most Pythonic, zero dependencies. Dataclass variant: typed, IDE-friendly. Polars variant: for data-science workflows with large datasets. The key point: change only the return type annotation on data(). Zero QML changes. Demo: QML Controls from Python

examples/controls_from_python/ — Slider, Labels and Button created entirely in Python

Load any QML type

load_qml_component() wraps built-in or custom QML types

Create & configure

.create() instantiates it — properties are plain Python attributes

Connect signals

QML signals connect directly to Python callables with .connect()

Build and animate a full QML UI without writing QML.

controls_from_python is the reverse direction: instead of Python data flowing into QML, Python creates and drives QML widgets. load_qml_component wraps any QML type — built-in controls or your own custom QML. .create() instantiates it. Properties are readable and writable as plain Python attributes. Signals connect to Python callables with .connect(). A QTimer drives the slider from Python — the whole UI is orchestrated in Python. Show in editor: examples/controls_from_python/main.py
