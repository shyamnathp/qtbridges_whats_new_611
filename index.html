New Features Overview

New Features

  1. QtBridge.Signal — Python signals that are real Qt signals
  2. Auto Properties — zero-boilerplate property generation
  3. Table Data — dataclass, dict/JSON, and Polars support
  4. load_qml_component — full UI control from Python

QtBridge.Signal

  • Define signals on plain Python classes — no QObject subclass needed
  • Internally creates a real PySide / Qt signal on the dynamic QMetaObject of QAIM wrapper
  • Works with .connect(), .disconnect(), and .emit()
  • Signals are accessible from both Python and QML

QtBridge.Signal — Example


from QtBridge import bridge_instance, qtbridge, Signal

class Counter:
    countChanged = Signal(int)   # ← declare a signal

    def __init__(self):
        self._count = 0

    @property
    def count(self) -> int:
        return self._count

    @count.setter
    def count(self, value: int):
        if self._count != value:
            self._count = value
            self.countChanged.emit(value)  # ← emit from Python

    def increment(self):
        self.count += 1

▸ QML can bind: onCountChanged: console.log(value)

Auto Properties

  • QtBridge scans __init__ for self.x = ... assignments
  • No more @property boilerplate for simple attributes
  • Opt-out per class (auto_properties=False) or per field (exclude_properties)

Table Data Support

Three ways to expose tabular data from Python to QML:

1. Dataclasses

Read data in Python, populate a @dataclass, pass to QML

2. list[dict] / JSON

Structured JSON is inherently tabular. Pass it directly

3. Polars DataFrame

Native DataFrame integration with database connectors

Table Data — Polars

Why Polars?

  • Fast — written in Rust backend
  • Native connectors: PostgreSQL, MySQL, SQLite, DuckDB, CSV…

Value Proposition (Confluence)

load_qml_component

  • Inspired by Slint's Python API
  • Load any QML type — custom .qml files or built-in QtQuick Controls
  • Create instances, set properties, connect signals. All from Python
  • Enables full UI control from Python via composition
  • QtBridge-Python is no longer just a backend data bridge. It's a complete Python-to-QML toolkit

Composition with Custom QML Types


from QtBridge import qtbridge, bridge_instance, load_qml_component

# Load a user-defined QML type
Person = load_qml_component("person.qml")

class Employee:
    def __init__(self, name: str, age: int, department: str):
        self.person = Person.create(name=name, age=age)
        self.department = department
        self.person.birthdayHappened.connect(self._on_birthday)

    def _on_birthday(self):
        print(f"{self.person.name} is now {self.person.age}!")

    def celebrate(self):
        self.person.celebrateBirthday()  # call QML function from Python

▸ Plain Python classes composing QML objects — signals, properties, and methods all work natively.

load_qml_component — Example


from QtBridge import load_qml_component, qtbridge

# Load a QtQuick Controls type
Label = load_qml_component(module="QtQuick.Controls", type_name="Label")

@qtbridge(qml_file="Main.qml")
def main(window):
    content = window.property("contentItem")

    # Create a Label entirely from Python
    heading = Label.create(text="Hello from Python!")
    heading.qobject.setProperty("parent", content)
    heading.qobject.setProperty("x", 20)
    heading.qobject.setProperty("y", 20)

    # Properties are readable and writable
    heading.text = "Updated from Python!"

    return [heading]
